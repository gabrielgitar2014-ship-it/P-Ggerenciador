3. ARQUIVOS MODIFICADOS: src/context/FinanceContext.jsx e src/components/tabs/BancosTab.jsx
   
   AÇÕES REALIZADAS:
   - Correção do erro de requisição PATCH 406 (PostgREST PGRST116): 
     Durante a edição de um cartão, a chamada ao Supabase utilizava o modificador `.single()`.
     Quando a edição é efetuada sobre um cartão pré-cadastrado no sistema (sem `user_id` 
     vinculado ao usuário atual) ou cuja atualização é bloqueada pelas políticas de RLS 
     (Row Level Security), a API retorna 0 linhas modificadas, o que lançava uma exceção 
     crítica devido à restrição do `.single()`.
   - Modificação efetuada: O modificador `.single()` foi substituído por uma consulta padrão
     (`.select()`). Em seguida, implementou-se uma verificação explícita do tamanho do array
     retornado.
   - Tratamento de Falha: Caso nenhuma linha seja atualizada (`data.length === 0`), uma
     exceção amigável e legível é lançada.
   - UI Aprimorada: O componente `BancosTab.jsx` foi ajustado para exibir o `error.message`
     específico no `alert`, comunicando ao usuário que ele não possui permissão para editar 
     os cartões-padrão do sistema, instruindo-o a criar um cartão novo caso necessário.
